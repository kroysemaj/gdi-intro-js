Class 3 - Objects & The Magical Magic of Functions

Objects

Objects let us store a collection of properties.


                  var objectName = {
                    propertyName: propertyValue,
                    propertyName: propertyValue,
                    ...
                  };

Properties can be any data type (Number, Strings, Boolean, Arrays, Functions, even other objects)


                  var aboutMe = {
                    hometown: 'Atlanta, GA',
                    hair: 'Auburn',
                    likes: ['knitting', 'code'],
                    birthday: {month: 10, day: 17}
                  };

Accessing Objects

You can retrieve values using "dot notation"


                  var aboutMe = {
                    hometown: 'Atlanta, GA',
                    hair: 'Auburn'
                  };
                  var myHometown = aboutMe.hometown;

Or using "bracket notation" using the property's key (like arrays)


                  var myHair = aboutMe['hair'];

Changing Objects

You can use dot or bracket notation to change properties


                  var aboutMe = {
                    hometown: 'Atlanta, GA',
                    hair: 'Auburn'
                  };
                  aboutMe.hair = 'blue';

Add new properties


                  aboutMe.gender = 'female';
                  aboutMe["gender"] = 'female';

Or delete them


                  delete aboutMe.gender;
                  delete aboutMe["gender"];

Arrays of Objects

Since arrays can hold any data type, they can also hold objects

                  
var myCats = [
  {name: 'Lizzie',
   age: 18},
  {name: 'Daemon',
   age: 1}
];

for (var i = 0; i < myCats.length; i++) {
  var myCat = myCats[i];
  console.log(myCat.name + ' is ' + myCat.age + ' years old.');
}
=> 'Lizzie is 18 years old.'
=> 'Daemon is 1 years old.'

More about Objects

Just like other data types, objects can be passed into functions:

                  
var lizzieTheCat = {
  age: 18,
  furColor: 'grey',
  likes: ['catnip', 'milk'],
  birthday: {month: 7, day: 17, year: 1996}
}

function describeCat(cat) {
  console.log('This cat is ' + cat.age + ' years old with ' +
      cat.furColor + ' fur.');
}

describeCat(lizzieTheCat);
=> 'This cat is 18 years old with grey fur.'

Object methods

Objects can also hold functions.


var lizzieTheCat = {
  age: 18,
  furColor: 'grey',
  meow: function() {
    console.log('meowww');
  },
  eat: function(food) {
    console.log('Yum, I love ' + food);
  },
};

Call object methods using dot notation:


lizzieTheCat.meow();
lizzieTheCat.eat('brown mushy stuff');

  • RegExp

More about functions!




Functions are the life blood of JavaScript, so we're going to spend a little more time with them to demonstrate their power and versitility.

In JavaScript, Functions are first-class objects!

That means they are objects and can be used in any of the same ways objects can.

  • Stored in variables
  • Passed as arguments to functions
  • Created within functions
  • Returned from a function

Stored in variables you say?!

Yep, and then called similarly to the way you would call a function declared normally

               
                  var allAboutThatBass = function() {
                    console.log('no treble');
                  };

                  allAboutThatBass();
                  => 'no treble'

Wait a tick, that function didn't have a name!



As it turns out naming a function isn't strictly required. This is called an Anonymous Function.


Most often, you'll do this when a function is going to be called in-place or when providing it as a callback (more on that in a minute).


The function does have a name, it was the variable name. Still, it's possible to declare functions without names. That's important to remember.

Functions can be passed as arguments to other functions?!


Yes! These are called Lambdas (impress your friends with that little gem).


This is one of the most powerful features in JavaScript. Until recently JS was one of the only popular langauges that could do this. In recent years these kinds of functions are popping in all kinds of languages like Python, Ruby(?), even Java

This can work a couple of different ways. Let's look at them both.

Passing a pre-defined function as an argument

                
var secondLyric = function() {
  return "when we're dancing on the ceiling";
};

function finishThisLyric(firstLyric, someFunction){
  console.log(firstLyric, someFunction());
}

finishThisLyric('oh what a feeling', secondLyric);

=> 'oh what a feeling'
=> 'when we're dancing on the ceiling'

Defining a function in-line.

                
function finishThisLyric(firstLyric, callback) {
  console.log(firstLyric);
  callback();
}

finishThisLyric("It's the", function(){
  console.log("eye of the tiger it's the thrill of the fight!");
});

=> "It's the"
=> "eye of the tiger it's the thrill of the fight!"

Let's Develop It

Let's use one of JavaScript's built in object methods Array.forEach() to demonstrate using inline functions.


Array.forEach() is an easier way to iterate over an array and do stuff. All you have to do is pass forEach() a function that does what you want it to do to each element in the array (which is passed to your function). No need to use a for loop!

Like so!

                
                  var rainbowColors = [
                    'red',
                    'orange',
                    'yellow',
                    'green',
                    'blue',
                    'indigo',
                    'violet'
                  ];

                  rainbowColors.forEach(function(color){
                    console.log(color);
                  });
                  => 'red, orange, yellow, green, blue, indigo, violet'

Consider these

                
                  var fiftyWays = {
                    jack: 'step out the back',

                  function waysToLeaveYourLove(fiftyWays) {
                    console.log(fiftyWays.jack)

                  function waysToLeaveYourLever(waysObject) {

                  waysToLeaveYourLover({

                    stan: 'make a new plan',
                    gus: 'hop on the bus'
                  });
